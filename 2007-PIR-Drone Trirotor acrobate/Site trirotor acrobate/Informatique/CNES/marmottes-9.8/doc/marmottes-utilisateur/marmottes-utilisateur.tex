\section{présentation}\label{sec:presentation}
\bibliotheque{marmottes} (Modélisation d'Attitude par Récupération des
Mesures d'Orientation pour Tout Type d'Engin
Spatial)\glossary{[MARMOTTES] : Modélisation d'Attitude par Récupération
des Mesures d'Orientation pour Tout Type d'Engin Spatial} est une
bibliothèque regroupant des fonctions de simulation de contrôle
d'attitude multimissions, c'est-à-dire dont le code n'est pas dédié à
une plate-forme particulière de satellite.

Cette bibliothèque regroupe des objets \emph{métier} destinés à être
utilisés dans des applications de mécanique spatiale qui peuvent soit
être auto-suffisantes, soit être elles-mêmes des bibliothèques plus
spécialisées ; il s'agit donc d'une bibliothèque de niveau
intermédiaire.

\begin{changebar}
Ce document décrit la version 9.8 de la bibliothèque.
\end{changebar}

\section{description du contexte}\label{sec:contexte}
Cette bibliothèque récupère les descriptions complètes des senseurs
dans des fichiers de ressources ; le paramétrage des fonctions de
simulation est donc très simple, il suffit de passer le nom du fichier
de ressources, les noms de senseurs, les valeurs des
consignes.\glossary{[consigne] : ce terme est employé dans ce document
avec deux sens différents selon le contexte~:
\begin{itemize}
\item attitude désirée pour le satellite (consigne d'attitude)
\item valeur produite par un des senseurs utilisés pour le contrôle ;
d'attitude, lorsque l'attitude désirée est atteinte (valeur de
consigne du senseur).
\end{itemize}}

Les fonctions simulent un pilotage parfait respectant ces consignes et
permettent entre autres de retourner l'attitude (sous forme de
quaternion) et le spin. La bibliothèque \bibliotheque{cantor} (qui est
d'ailleurs utilisée par \bibliotheque{marmottes}) fournit les
fonctions de calcul liées aux quaternions dont peut avoir besoin le
programme principal.

La bibliothèque est écrite en \langage{c++} et dispose d'une interface
\langage{c++}, d'une interface \langage{c}, et d'une interface
\langage{fortran} (les fonctions de calcul de quaternions de
\bibliotheque{cantor} disposent des mêmes interfaces).

La liste ordonnée des bibliothèques à spécifier à l'éditeur de liens
est la suivante :
\begin{center}
\texttt{-lmarmottes} \texttt{-lcantor} \texttt{-lclub}
\end{center}

Il est indispensable de faire l'édition de liens avec le compilateur
\langage{c++} qui a été utilisé pour compiler la bibliothèque car
celle-ci utilise le mécanisme des exceptions qui nécessite un support
de la part de l'éditeur de liens. Si le programme appelant est en
\langage{fortran} ou en \langage{fortran 90}, il faut spécifier
explicitement les bibliothèques fortran (par exemple avec la version
4.2 du compilateur fortran 77 SUN, il faut ajouter \texttt{-lF77}
\texttt{-lM77} \texttt{-lsunmath}).

Cette note décrit les fichiers de ressources des senseurs et toutes
les routines de bibliothèque. Les routines liées aux quaternions sont
décrites dans la documentation de \bibliotheque{cantor}
\ref{ref:cantor}. Les principes de résolution et les algorithmes
utilisés sont décrits en détail dans la documentation mathématique de
\bibliotheque{marmottes}~\ref{ref:marmottes-math}.
