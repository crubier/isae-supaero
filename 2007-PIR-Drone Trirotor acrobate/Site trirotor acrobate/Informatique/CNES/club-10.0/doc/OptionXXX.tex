\subsection{classes dérivées de OptionBase}\label{sec:OptionXXX}

\subsubsection*{description}\label{sec:OptionXXX-desc}
Les classes dérivées de la classe OptionBase modélisent les divers
types d'options dont on peut avoir besoin dans la ligne de commande
d'une application. Ces options sont différenciées par les arguments
qu'elles attendent, comme le montre la
table~\ref{tab:derivees-OptionBase}.

\subsubsection*{interface publique}\label{sec:OptionXXX-int}
\begin{verbatim}
#include "club/OptionsAppli.h"
\end{verbatim}

Le fichier de déclaration de la classe OptionsAppli (qui déclare la
classe d'analyseur qui regroupera les instances d'options) inclut
directement tous les fichiers d'en-tête des diverses options.

Les méthodes publiques de toutes ces classes sont héritées (ou
redéfinies) à partir des méthodes de la classe OptionBase, elles sont
décrites dans la table~\ref{tab:OptionBase-met-pub},
page~\pageref{tab:OptionBase-met-pub}.

Les méthodes qui ne sont pas applicables à un type d'option (par
exemple lire une valeur réelle pour une option attendant un tableau de
chaînes de caractères) génèrent systématiquement un message d'erreur
explicitant le conflit de type.

Les seules méthodes nécessitant une description sont les
constructeurs, qui sont spécifiques à chaque classe dérivée :

\subsubsection*{implantation}\label{sec:OptionXXX-impl}
La classe OptionSansValeur n'a aucun attribut supplémentaire par
rapport à sa classe de base.

Les options sans tableau sont dérivées des classe avec tableau, ce
sont des spécialisations pour la cas où la dimension du tableau est 1.
